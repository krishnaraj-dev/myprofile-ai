# Security Policy

## Supported Versions
1. Latest `main` branch is supported.

## Reporting a Vulnerability
1. Create a private report via GitHub Security Advisories or contact the maintainer directly.
2. Include reproduction steps, impact, and any suggested fix.

## Remediation Policy
1. Critical: fix within 7 days.
2. High: fix within 14 days.
3. Medium: fix within 30 days.
4. Low: fix within 90 days.

Dependency audits are tracked via `npm audit` and manual review before releases.
